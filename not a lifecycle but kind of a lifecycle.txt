1. Object created
2. Function/constructor called
3. Reference or pointer used
4. Temporary created or copied
5. Function ends
6. Object destroyed


reference = same person, different nickname
pointer = a sticky note with an address
constructor = filling out a new ID card
destructor = shredding the ID card
vector = a row of boxes containing copies or moved objects

Before reading any line, ask:

“Is this operating on the original object, a copy, or an address to it?”
